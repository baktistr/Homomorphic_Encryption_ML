# HE Fraud Detection Dashboard — Implementation Plan

## Overview

A Flask-based interactive dashboard that demonstrates privacy-preserving fraud detection using Homomorphic Encryption (HE). The dashboard runs **both** a normal (plaintext) ML pipeline and an HE (CKKS-encrypted) pipeline side-by-side for **four ML algorithms**, allowing users to compare accuracy, latency, ciphertext overhead, and privacy guarantees.

### Models Supported

| # | Model | Representation | HE Complexity |
|---|-------|---------------|---------------|
| 1 | Ridge Score | 6D base | Low (weighted sum) |
| 2 | Logistic Regression | 6D base | Low (weighted sum) |
| 3 | Linear SVC | 6D base | Low (weighted sum) |
| 4 | Degree-2 Polynomial LogReg | Poly-expanded 6D | Medium (larger vector) |

### Feature Vector (6D Base)

```
amount_scaled, oldbalanceOrg_scaled, oldbalanceDest_scaled,
deltaOrig_scaled, deltaDest_scaled, is_transfer
```

3. Visual lock/unlock icons (🔒/🔓) distinguish encrypted vs plaintext sections
4. Workflow animation on landing page: `Client (encrypt) → Server (compute on ciphertext) → Client (decrypt)`

## Implementation Phases

### Phase 1: Foundation & Backend Pipeline (Est. ~3-4 hours)

**Goal**: Flask app skeleton + all inference logic working in Python, no frontend yet.

#### Tasks

1. **Create `requirements.txt`**
   ```
   flask>=3.0
   numpy
   pandas
   scikit-learn
   scipy
   tenseal
   ```

2. **Create `config.py`**
   - Paths to artifacts directory
   - CKKS parameter configurations (3 configs from notebook: `fast_8192`, `balanced_16384`, `secure_16384`)
   - Feature order constant
   - Upload folder config

3. **Create `pipeline/preprocessing.py`**
   - `load_scaler()` — load saved StandardScaler
   - `engineer_features(raw_dict) -> np.ndarray` — take a raw transaction dict (type, amount, balances), compute deltaOrig/deltaDest/is_transfer, scale, return 6D vector
   - `engineer_features_batch(df) -> np.ndarray` — same for DataFrame
   - `validate_input(raw_dict) -> (bool, errors)` — input validation

4. **Create `pipeline/models.py`**
   - `load_all_models()` — returns dict of `{model_name: {weights, bias, representation, input_dim}}`
   - Loads Ridge, LogReg, LinearSVC (all 6D), and Poly2 LogReg (expanded)
   - `get_poly2_expansion(X_6d) -> X_expanded` — polynomial feature expansion matching notebook

5. **Create `pipeline/plaintext_inference.py`**
   - `predict_single(X, model_spec) -> {score, label, time_ms}`
   - `predict_batch(X_all, model_spec) -> [{score, label, time_ms}, ...]`
   - Score = `X @ weights + bias`, label = `score > 0` threshold

6. **Create `pipeline/encrypted_inference.py`**
   - `build_ckks_context(config) -> ts.Context`
   - `predict_single_he(X, model_spec, ckks_config) -> {score, label, encrypt_ms, infer_ms, decrypt_ms, ciphertext_bytes, abs_error}`
   - `predict_batch_he(X_all, model_spec, ckks_config) -> [...]`
   - Graceful fallback if TenSEAL not installed

7. **Create `app.py`** (routes only, render placeholder templates)
   - `GET /` — landing
   - `GET /single` — form, `POST /single` — results
   - `GET /batch` — upload form, `POST /batch` — process CSV
   - `GET /comparison` — pre-computed comparison

#### Verification
- Run each pipeline function from a Python script with a sample from `he_eval_scaled_features_6d.csv`
- Confirm plaintext scores match notebook reference scores
- Confirm HE scores match within tolerance

## Key Design Decisions

1. **Flask over Streamlit/Dash**: Project requirement specifies Flask; also gives full control over the privacy-preserving UI (server vs client panels)
2. **Chart.js over Matplotlib**: Interactive browser charts, no server-side image generation
3. **Pre-computed comparison page**: Avoids re-running HE inference on every page load (cache on first request)
4. **Graceful TenSEAL fallback**: Dashboard remains fully functional for plaintext demo even without TenSEAL
5. **No real client-server split**: Both pipelines run on the same Flask server for simplicity; the UI *simulates* the client/server boundary visually
