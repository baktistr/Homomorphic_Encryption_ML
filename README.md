# Privacy-Preserving Fraud Detection with Homomorphic Encryption

A Flask-based dashboard that demonstrates fraud detection on financial transactions using **Homomorphic Encryption (HE)**. The system runs both a normal (plaintext) ML pipeline and a CKKS-encrypted pipeline side-by-side for four models, allowing direct comparison of accuracy, latency, and privacy guarantees.

**CMU EPS Project** — Bakti Satria Adhityatama, Elijah Zhao, Aditi Ashok Kurutala

## Models

| Model | Input | HE Cost |
|-------|-------|---------|
| Ridge Score | 6D scaled | Low |
| Logistic Regression | 6D scaled | Low |
| Linear SVC | 6D scaled | Low |
| Poly2 Logistic Regression | 27D expanded | Medium |

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

Dependencies: Flask, NumPy, Pandas, scikit-learn, SciPy, TenSEAL

### 2. Download the PaySim dataset

Download from [Kaggle](https://www.kaggle.com/datasets/ealaxi/paysim1) and place the CSV in `data/`:

```
data/PS_20174392719_1491204439457_log.csv
```

### 3. Generate artifacts

Run the baseline training notebook to produce model artifacts:

```bash
jupyter notebook fraud_baseline_6d.ipynb
# Run all cells
```

This creates the `artifacts/` directory with trained model weights, scaler, and evaluation subsets.

### 4. Run the dashboard

```bash
python app.py
```

Open **http://localhost:5000** in your browser.

## Dashboard Pages

- **Home** (`/`) — Project overview, workflow diagram, model descriptions
- **Single Transaction** (`/single`) — Enter one transaction, see normal vs HE inference for all 4 models
- **Batch Upload** (`/batch`) — Upload a CSV, get aggregated metrics and timing comparison
- **Comparison** (`/comparison`) — Pre-computed results on 128-row eval subset with charts

## How It Works

1. **Client encrypts** — Transaction features are scaled into a 6D vector and encrypted using CKKS (TenSEAL). The secret key never leaves the client.
2. **Server computes** — The server performs `X · w + b` directly on the encrypted vector using homomorphic operations. It never sees plaintext.
3. **Client decrypts** — The encrypted score is returned and decrypted locally. The fraud/not-fraud threshold is applied client-side.

## CKKS Parameter Configurations

| Config | poly_modulus_degree | Use Case |
|--------|-------------------|----------|
| `fast_8192` | 8192 | Fast demo |
| `balanced_16384` | 16384 | Good balance of speed/security |
| `precise_16384` | 16384 | Higher precision |
