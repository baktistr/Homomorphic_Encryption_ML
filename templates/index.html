How It Works
🔓

Client — Encrypt

Transaction features are converted to a 6D vector and encrypted locally using CKKS. The secret key never leaves the client.

→
🔒

Server — Compute

The server computes X · w + b directly on the encrypted vector. It never sees plaintext features or scores.

→
🔓

Client — Decrypt

The encrypted fraud score is returned and decrypted locally. The final fraud/not-fraud decision is made client-side.

Models

Ridge Score

6D input

Weighted sum baseline. Low HE cost — single dot product on 6 encrypted features.

Logistic Regression

6D input

Logistic scoring with sigmoid applied after decryption. Same HE cost as Ridge.

Linear SVC

6D input

Support vector classifier decision function. Identical HE structure to Ridge/LogReg.

Poly2 Logistic Regression

27D input

Degree-2 polynomial expansion of 6D features (27 terms). Higher HE cost due to larger vector.
